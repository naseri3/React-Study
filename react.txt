// react란?
: React는 사용자 화면(UI)을 만들기 위한 JavaScript 라이브러리다.
  "웹 화면을 효율적으로 만들고 관리하기 위한 도구"이다

// React의 핵심 개념
1. 컴포넌트 기반
- 화면을 작은 UI조각(컴포넌트)으로 나눔
- 버튼, 카드, 헤더, 로그인폼 등
- 재사용 가능 + 유지보수 쉬움

2. 선언형 UI
{isLogin ? <Logout/> : <Login/>}
- "이 상태면 이런 화면"이라고 선언만 함
- 어떻게 바뀌는지는 React가 처리

3. 상태(state) 중심
- 화면은 상태(state)의 결과물
- 상태가 바뀌면 화면도 자동 변경
- DOM 직접 조작 X
=> React = 상태 -> 화면


// React는 프레임워크가 아니라 라이브러리다
- React는 UI만 담당
- 라우팅, 서버 통신, 상태 관리 등은 선택
- 필요한 것만 골라서 사용
- 자유도가 높음
- 대신 구조 설계는 개발자 몫


// React가 잘 맞는 경우
- 화면 변화가 많은 서비스
- 로그인 / 필터 / 모달 / 데이터 리스트
- SPA(페이지 새로고침 없는 웹)

// 컴포넌트란?
: React 컴포넌트(Component)는 화면을 구성하는 하나의 독립된 UI조각이다.
  웹 페이지를 큰 HTML 덩어리로 만들지 않고, 작은 부품 단위로 쪼개서 만드는 방식이라고 보면 됨


// 왜 컴포넌트로 나누는가?
기존 방식(HTML / JS) :
- 한 페이지에 코드가 계속 늘어남
- 수정할 때 어디가 영향을 받는지 파악이 어려움
- 같은 UI를 다시 만들기 힘듦

React 방식:
- 버튼, 카드, 헤더, 리스트 등을 각각 컴포넌트로 분리
- 필요한 곳에 재사용
- 수정하면 그 컴포넌트만 영향
=> 유지보수 + 재사용 + 가독성 때문에 컴포넌트를 사용


// 컴포넌트는 결국 함수다.
  function Header() {
    return <h1> 안녕하세요. react 시작 </h1>;
  }
  JSX(HTML처럼 보이는 문법)를 return한다
  즉, "화면을 반환하는 함수" = 컴포넌트


// 컴포넌트의 핵심 규칙
1. 이름은 반드시 대문자로 시작
  - Header, LoginForm, CardItem
2. 하나의 UI 역할만 담당
  - 버튼 컴포넌트 = 버튼만
  - 카드 컴포넌트 = 카드만
3. return은 하나의 덩어리
  - div 하나로 감싸거나 Fragment 사용


// 컴포넌트로 나누는 기준

- 특정 페이지에서만 쓰는 안내 문구 X (나중에 분리해도 됨)

// props
: props는 부모 컴포넌트가 자식 컴포넌트에게 데이터를 전달하기 위한 읽기 전용 값이며, 컴포넌트의 재사용성과 역할 분리를 위해 사용된다.
- properties의 줄임말
- 컴포넌트 간 데이터 전달 통로
- 부모 -> 자식 방향으로만 전달(단방향)
- 자식 컴포넌트에서는 수정할 수 없음(read-only)
: "props는 컴포넌트에 내려주는 설정값"이라고 생각하면 딱 좋음


// props 기본 사용법
- 부모 컴포넌트
function App() {
  return <Welcome name="나슬" />;
}

- 자식 컴포넌트
function Welcome(props) {
  return <h1>안녕하세요, {props.name}</h1>;
}

- 구조 분해 할당으로 props 받기
function Welcome({ name }) {
  return <h1>안녕하세요, {name}</h1>;
}

- props로 여러 값 전달하기
<Profile 
  name="나슬"
  age={30}
  job="Frontend"
/>
function Profile({ name, age, job }) {
  return (
    <p>
      {name} / {age} / {job}
    </p>
  );
}

- props는 수정이 안 됨
function Box({ size }) {
  size = 100; // ❌ 금지
}
** React는 props를 불변값으로 관리
** 수정하려면 → state 사용

- props로 함수도 전달 가능
function App() {
  const handleClick = () => {
    alert("클릭!");
  };

  return <Button onClick={handleClick} />;
}
function Button({ onClick }) {
  return <button onClick={onClick}>클릭</button>;
}
** 부모가 로직 관리
** 자식은 UI 역할
